## 3. テストシナリオ詳細

### AUTH-001: 新規ユーザーがサインアップできる

**前提条件**
- 未ログイン状態

**手順**
1. トップページにアクセス
2. 「サインアップ」ボタンをクリック
3. メールアドレスとパスワードを入力
4. 「登録」ボタンをクリック

**期待結果**
- 登録完了メッセージが表示される
- ダッシュボードにリダイレクトされる

## 4. 優先度の定義

| 優先度 | 説明 |
|--------|------|
| P0 | 必須：クリティカルパス、リリースブロッカー |
| P1 | 重要：主要機能、エラーハンドリング |
| P2 | 推奨：エッジケース、UX向上 |

## 5. ステータスの定義

| ステータス | 説明 |
|-----------|------|
| - | 未実装 |
| 実装済 | E2Eテストコード作成済み |
| Pass | テスト成功 |
| Fail | テスト失敗（要修正） |
